# 房源类型转码和图片问题修复记录

## 问题描述

用户报告了两个问题：

1. **房源类型转码问题**: 民宿卡片没有翻译出房源类型的转码，显示的是英文代码而不是中文名称
2. **图片 403 错误**: 更新卡片后有很大一部分房源图片加载不出来，控制台显示 403 Forbidden 错误

## 问题原因分析

### 1. 房源类型转码问题

在组件抽离重构时，将房源类型转码逻辑留在了`Home.vue`的`getHomestayTypeText`方法中，但`HomestayCard`组件没有使用这个逻辑，而是直接显示原始的类型代码。

**原始问题代码**:

```js
// HomestayCard.vue 中直接使用原始代码
if (props.homestay.propertyType || props.homestay.type) {
  features.push(props.homestay.propertyType || props.homestay.type);
}
```

### 2. 图片 403 错误问题

图片 URL 拼接逻辑有问题，可能导致图片路径不正确，从而出现 403 访问被拒绝的错误。

**原始问题代码**:

```js
// 简单的URL拼接可能导致格式错误
return `/api${image.startsWith("/") ? "" : "/"}${image}`;
```

## 修复方案

### 1. 房源类型转码修复

#### 步骤 1: 在 HomestayCard 组件中添加类型转码逻辑

```js
// 添加props接收房源类型数据
interface Props {
    homestay: any
    iconSize?: number
    showCarousel?: boolean
    homestayTypes?: any[]  // 新增：房源类型数据
}

// 添加类型转码功能
const getHomestayTypeText = (typeCode: string | undefined): string => {
    if (!typeCode) return '未知类型'

    // 从传入的房源类型列表中查找
    const foundType = props.homestayTypes.find(t => t.code === typeCode)

    if (foundType) {
        return foundType.name || typeCode
    }

    // 如果找不到匹配项，返回原始代码
    return typeCode
}

// 更新features计算属性使用类型转码
const features = computed(() => {
    // ... other features

    // 使用类型转码功能
    if (props.homestay.propertyType || props.homestay.type) {
        const typeCode = props.homestay.propertyType || props.homestay.type
        features.push(getHomestayTypeText(typeCode))
    }

    // ... rest
})
```

#### 步骤 2: 在 Home.vue 中传递房源类型数据

```vue
<!-- 推荐民宿区域 -->
<HomestayCard
  v-for="homestay in homestays"
  :key="homestay.id"
  :homestay="homestay"
  :homestay-types="fetchedTypes"
  @click="viewHomestayDetails(homestay.id)"
/>

<!-- 最新上架区域 -->
<HomestayCard
  v-for="homestay in activeHomestays"
  :key="homestay.id"
  :homestay="homestay"
  :homestay-types="fetchedTypes"
  @click="viewHomestayDetails(homestay.id)"
/>
```

### 2. 图片 URL 处理优化

#### 修复前的问题

- 简单的字符串拼接可能导致 URL 格式错误
- 没有统一的图片 URL 处理逻辑
- 对不同类型的图片数据结构处理不当

#### 修复后的解决方案

```js
const images = computed(() => {
  const defaultImage = "https://picsum.photos/300/200";

  // 图片URL处理函数 - 统一处理各种图片URL格式
  const processImageUrl = (imageUrl: string): string => {
    if (!imageUrl) return defaultImage;

    // 如果是完整的HTTP/HTTPS URL，直接返回
    if (imageUrl.startsWith("http://") || imageUrl.startsWith("https://")) {
      return imageUrl;
    }

    // 如果是相对路径，构建完整的后端API URL
    const cleanUrl = imageUrl.startsWith("/") ? imageUrl : `/${imageUrl}`;
    return `http://localhost:8080${cleanUrl}`;
  };

  // 处理图片数组
  if (
    props.homestay.images &&
    Array.isArray(props.homestay.images) &&
    props.homestay.images.length > 0
  ) {
    return props.homestay.images
      .map((image: any) => {
        if (typeof image === "string") {
          return processImageUrl(image);
        }
        // 处理对象类型的图片数据，支持url和path字段
        return processImageUrl(image?.url || image?.path) || defaultImage;
      })
      .filter(Boolean); // 过滤空值
  }

  // 处理封面图片
  if (props.homestay.coverImage) {
    return [processImageUrl(props.homestay.coverImage)];
  }

  return [defaultImage];
});
```

- ✅ 支持多种图片数据格式（字符串、对象）

## 技术改进点

### 1. 组件数据传递优化

通过 props 向子组件传递必要的数据，而不是在子组件中重复实现相同逻辑，提高了代码复用性。

### 2. 错误处理增强

- 添加了空值检查
- 提供默认值 fallback
- 统一的 URL 格式化处理

### 3. 数据结构兼容性

支持多种图片数据结构：

- 字符串类型: `"image.jpg"`
- 对象类型: `{url: "image.jpg"}` 或 `{path: "image.jpg"}`
