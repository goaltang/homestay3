# 支付宝页面跳转调试指南

## 问题现象

用户点击支付宝支付后，页面没有跳转到支付宝支付页面。

## 调试步骤

### 1. 前端调试

打开浏览器开发者工具（F12），在 Console 标签页查看以下信息：

#### 关键日志：

```javascript
// 查看支付宝返回的数据
支付宝返回的paymentUrl: [HTML内容]

// 查看表单查找结果
找到支付表单，准备提交

// 或者错误信息
表单提交失败: [错误详情]
尝试在新窗口打开支付页面
新窗口打开失败: [错误详情]
```

#### 检查 Network 标签：

1. 找到 `/api/payment/{orderId}/create?method=alipay` 请求
2. 查看 Response 内容，确认：
   - `success: true`
   - `paymentUrl` 字段包含 HTML 表单
   - 没有网络错误

### 2. 后端调试

查看后端控制台日志：

#### 关键日志：

```
调用支付宝页面跳转支付，订单ID: {orderId}
支付宝页面支付请求参数: {...}
支付宝页面支付表单生成成功，订单号: {outTradeNo}
```

#### 如果看到错误：

```
支付宝创建页面支付订单异常，尝试次数: X/X, 错误: Read timed out
```

说明是网络超时问题。

### 3. 常见问题及解决方案

#### 问题 1: 网络超时

**现象**: 后端日志显示 "Read timed out"
**解决**:

- 检查网络连接
- 确认支付宝沙箱环境可访问
- 增加超时时间配置

#### 问题 2: 浏览器阻止弹窗

**现象**: 前端显示 "新窗口打开失败"
**解决**:

- 允许浏览器弹窗
- 使用当前窗口跳转而不是新窗口

#### 问题 3: HTML 表单格式错误

**现象**: 前端找不到 form 元素
**解决**:

- 检查支付宝返回的 HTML 格式
- 验证支付宝配置参数

### 4. 手动测试方法

#### 方法 1: 直接 API 测试

在浏览器中访问（需要登录 token）：

```
POST http://localhost:8080/api/payment/{orderId}/create?method=alipay
```

#### 方法 2: 模拟支付测试

使用系统的"模拟支付成功"功能验证支付流程。

#### 方法 3: 降级到二维码支付

临时使用微信支付或其他二维码支付方式。

### 5. 临时解决方案

如果页面跳转持续失败，可以：

1. **使用二维码扫码支付**：

   - 选择微信支付
   - 扫码完成支付

2. **使用模拟支付**：

   - 开发环境下点击"模拟支付成功"
   - 直接完成订单

3. **手动处理**：
   - 联系系统管理员
   - 线下确认支付

### 6. 配置检查清单

确认以下配置正确：

- [ ] 支付宝 APPID: 9021000149671635
- [ ] 应用私钥已正确配置
- [ ] 支付宝公钥已正确配置
- [ ] 网关 URL: https://openapi.alipaydev.com/gateway.do
- [ ] 回调 URL 配置正确
- [ ] 网络连接正常

### 7. 紧急处理

如果支付功能完全不可用：

1. 暂时禁用支付宝支付选项
2. 引导用户使用其他支付方式
3. 记录问题详情，联系技术支持

## 预期正常流程

正常情况下应该看到：

1. 用户点击"立即支付"
2. 前端显示"正在跳转到支付页面..."
3. 浏览器自动跳转到支付宝官方支付页面
4. 用户在支付宝页面完成支付
5. 支付完成后自动返回民宿系统

## 系统改进

### 自动降级机制

- 后端检测到页面跳转失败时，自动切换到二维码支付
- 避免用户看到错误信息，提供备用支付方式

### 前端重试功能

- 新增"重新生成支付"按钮
- 用户可以多次尝试生成支付
- 提供友好的使用说明

### 用户指导

- 显示支付说明和操作指南
- 推荐备用支付方式
- 提供技术支持联系方式
