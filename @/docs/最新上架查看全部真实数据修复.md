# 最新上架"查看全部"真实数据修复记录

## 问题描述

用户报告点击"最新上架"的"查看全部"按钮时，显示的是模拟数据而不是真实的后端数据。

## 问题原因分析

### 1. 错误的 API 调用

`HomestayListView.vue`在处理`status=ACTIVE`查询参数时，使用了错误的 API 端点：

```js
// 错误的API调用
response = await request.get("/api/homestays/active");
```

这个端点`/api/homestays/active`在后端并不存在，导致 API 调用失败，系统降级使用模拟数据。

### 2. 未使用正确的 API 函数

在`homestay.ts` API 文件中已经定义了正确的`getActiveHomestays`函数：

```js
export function getActiveHomestays(params?: {
  page?: number,
  size?: number,
  featured?: boolean,
}) {
  return request({
    url: "/api/homestays",
    method: "get",
    params: {
      ...params,
      status: "ACTIVE",
    },
  });
}
```

但是`HomestayListView.vue`没有导入和使用这个函数。

### 3. 缺少房源类型数据

`HomestayCard`组件需要房源类型数据来进行类型转码，但`HomestayListView.vue`没有传递这些数据。

## 修复方案

### 1. 导入正确的 API 函数

```js
// 添加API函数导入
import { getActiveHomestays, getHomestayTypes } from "@/api/homestay";
```

### 2. 使用正确的 API 调用

```js
// 修复前：错误的API调用
response = await request.get("/api/homestays/active");

// 修复后：使用正确的API函数
response = await getActiveHomestays({
  page: currentPage.value - 1,
  size: pageSize.value,
});
```

### 3. 添加房源类型数据支持

```js
// 添加房源类型数据
const homestayTypes = ref<any[]>([]);

// 获取房源类型数据
const loadHomestayTypes = async () => {
    try {
        const response = await getHomestayTypes();
        homestayTypes.value = response || [];
    } catch (error) {
        console.error('获取房源类型失败:', error);
    }
};

// 在组件挂载时加载房源类型
onMounted(() => {
    initializeFromRoute();
    loadHomestayTypes();  // 新增
    loadHomestays();
    console.log('路由查询参数:', route.query);
});
```

### 4. 传递房源类型数据给 HomestayCard

```vue
<!-- 修复前 -->
<HomestayCard :homestay="homestay" />

<!-- 修复后 -->
<HomestayCard :homestay="homestay" :homestay-types="homestayTypes" />
```

## 修复效果

### API 调用优化

- ✅ **正确的端点**: 使用`/api/homestays?status=ACTIVE`而不是不存在的`/api/homestays/active`
- ✅ **分页支持**: 支持分页参数，可以加载更多数据
- ✅ **错误处理**: 使用 API 函数中内置的错误处理和降级机制

### 数据展示改进

- ✅ **真实数据**: 显示后端数据库中的真实最新上架房源
- ✅ **类型转码**: 房源类型显示中文名称而不是代码
- ✅ **数据一致性**: 与首页的最新上架区域保持数据一致

### 用户体验提升

- ✅ **数据准确性**: 用户看到的是实时的最新上架房源
- ✅ **分页功能**: 可以浏览更多最新上架的房源
- ✅ **加载性能**: 减少不必要的错误请求

## 技术改进点

### 1. API 设计一致性

使用统一的 API 函数而不是直接调用 request，确保：

- 统一的错误处理
- 一致的参数格式
- 更好的代码维护性

### 2. 数据传递优化

- 通过 props 向子组件传递必要的数据
- 避免子组件重复请求相同的数据
- 提高组件复用性

### 3. 错误降级机制

API 函数内置了多层降级机制：

```js
// 1. 尝试无版本路径 /api/homestays
// 2. 如果失败，尝试v1版本 /api/v1/homestays
// 3. 确保系统健壮性
```
