# 组件抽离重构记录

## 重构背景

Home.vue 文件已达到近 2000 行代码，代码臃肿，维护困难。需要将搜索相关功能拆分为独立组件，提高代码可维护性。

## 新创建的组件

### 1. SearchBar.vue - 搜索栏组件

**位置**: `homestay-front/src/components/SearchBar.vue`

**功能包含**:

- 目的地选择（地区级联选择器）
- 日期选择（入住/退房日期）
- 房客人数选择
- 关键词输入
- 搜索/重置按钮
- 快捷搜索建议

**对外接口**:

```typescript
// Props
interface Props {
  loading?: boolean;
}

// Events
const emit = defineEmits<{
  search: [params: any];
  reset: [];
}>();

// Expose
defineExpose({
  searchParams,
  resetParams: handleReset,
});
```

### 3. FilterBar.vue - 筛选栏组件

**位置**: `homestay-front/src/components/FilterBar.vue`

**功能包含**:

**对外接口**:

## Home.vue 重构改进

### 模板简化

**重构前**: 150+ 行复杂的搜索栏和筛选栏模板代码
**重构后**: 简洁的组件引用

```vue
<template>
  <div class="home-container">
    <!-- 搜索栏组件 -->
    <SearchBar :loading="loading" @search="handleSearch" @reset="handleReset" />

    <!-- 筛选栏组件 -->
    <FilterBar
      v-if="pageState !== 'browse'"
      :property-types="fetchedTypes"
      :grouped-amenities="groupedAmenities"
      :amenities-loading="amenitiesLoading"
      :selected-property-type="searchParams.propertyType"
      @property-type-change="handlePropertyTypeChange"
      @filters-change="handleFiltersChange"
      @filters-reset="handleFiltersReset"
    />

    <!-- 民宿网格 -->
    <div v-else class="homestay-grid">
      <HomestayCard
        v-for="homestay in homestays"
        :key="homestay.id"
        :homestay="homestay"
        @click="viewHomestayDetails(homestay.id)"
      />
    </div>
  </div>
</template>
```

### JavaScript 简化

**移除的代码**:

- 搜索栏相关状态管理 (~50 行)
- 地区选择器配置和数据 (~30 行)
- 客人数量管理逻辑 (~40 行)
- 搜索建议相关逻辑 (~20 行)
- 日期验证逻辑 (~15 行)
- 搜索栏交互方法 (~80 行)
- 筛选相关方法 (~30 行)

**新增的方法**:

```typescript
// 搜索栏组件事件处理
const handleSearch = (params: any) => {
  Object.assign(searchParams, params);
  searchHomestays();
};

const handleReset = () => {
  resetAndRefresh();
};

// 筛选栏组件事件处理
const handlePropertyTypeChange = (typeName: string | null) => {
  searchParams.propertyType = typeName;
  searchHomestays();
};

const handleFiltersChange = (filters: any) => {
  searchParams.minPrice = filters.minPrice;
  searchParams.maxPrice = filters.maxPrice;
  searchParams.amenities = filters.amenities;
  searchHomestays();
};

const handleFiltersReset = () => {
  searchParams.minPrice = null;
  searchParams.maxPrice = null;
  searchParams.amenities = [];
  searchHomestays();
};
```

## 架构优势

### 1. 职责分离

- **SearchBar**: 负责搜索条件收集
- **FilterBar**: 负责筛选条件管理
- **Home**: 负责整体状态协调和数据获取

### 2. 组件复用

- SearchBar 可在其他页面复用
- FilterBar 可独立使用
- 降低代码重复率

### 3. 易于维护

- 每个组件职责单一
- 问题定位更容易
- 修改影响范围可控

### 4. 易于测试

- 组件可独立测试
- Mock 依赖更简单
- 测试覆盖率更高

## 技术细节

### 组件通信方式

- **父到子**: Props 传递数据
- **子到父**: Events 传递事件
- **状态同步**: expose 暴露子组件方法

### 样式管理

- 各组件独立管理样式
- 使用 scoped 避免样式污染
- 保持设计一致性

### TypeScript 支持

- 完整的类型定义
- Props 和 Events 类型安全
- 更好的开发体验
