# 房源审核机制改进 - 第二阶段实施计划

## 📋 第二阶段目标

在第一阶段基础架构的基础上，完善用户体验和功能完整性，包括前端界面优化、通知机制实现和权限控制完善。

## 🎯 核心功能模块

### 1. 前端审核界面优化 (1-2 周)

#### 1.1 管理员审核工作台

- **增强审核界面**

  - 添加审核记录查看功能
  - 显示房源提交历史和当前审核员
  - 集成 IP 地址和操作来源显示
  - 支持审核备注和详细理由输入

- **审核操作优化**

  - 分配审核员功能
  - 支持审核员工作量查看
  - 添加审核时限提醒
  - 状态转换可视化显示

- **数据统计面板**
  - 待审核房源数量实时显示
  - 各状态房源分布图表
  - 审核员工作效率统计
  - 审核通过率分析

#### 1.2 房东端审核状态跟踪

- **状态跟踪界面**

  - 房源状态实时展示
  - 审核进度条显示
  - 审核历史时间线
  - 下一步操作指导

- **交互优化**
  - 状态变更通知
  - 重新提交简化流程
  - 拒绝原因详细展示
  - 修改建议突出显示

#### 1.3 审核历史可视化

- **历史记录界面**
  - 完整审核链路展示
  - 操作人员和时间详细记录
  - IP 地址和设备信息显示
  - 导出审核报告功能

### 2. 通知机制实现 (1 周)

#### 2.1 站内消息系统

- **消息实体设计**

  - 消息类型分类(审核、状态变更、系统通知)
  - 优先级设置(紧急、重要、普通)
  - 读取状态追踪
  - 消息有效期管理

- **消息推送逻辑**
  - 审核状态变更自动推送
  - 房源提交确认消息
  - 审核员任务分配通知
  - 系统维护公告

#### 2.2 邮件通知功能

- **邮件模板设计**

  - 审核通过邮件模板
  - 审核拒绝邮件模板
  - 重新提交提醒模板
  - 系统通知模板

- **邮件发送机制**
  - 异步邮件发送队列
  - 发送失败重试机制
  - 邮件发送状态追踪
  - 用户邮件偏好设置

#### 2.3 短信提醒集成

- **短信场景设计**
  - 重要状态变更短信提醒
  - 审核超时预警短信
  - 紧急通知短信
  - 验证码短信

### 3. 权限控制完善 (1 周)

#### 3.1 细粒度权限管理

- **权限角色细分**

  - 高级审核员：可审核所有房源
  - 普通审核员：只能审核分配的房源
  - 审核主管：可分配审核员和查看统计
  - 系统管理员：所有权限

- **权限控制实现**
  - 基于角色的访问控制(RBAC)
  - API 接口权限验证
  - 前端页面权限控制
  - 操作按钮权限显示

#### 3.2 审核员分配机制

- **自动分配算法**

  - 基于工作量的智能分配
  - 根据专业领域分配
  - 考虑审核员在线状态
  - 负载均衡优化

- **手动分配功能**
  - 管理员手动指定审核员
  - 审核员工作量查看
  - 分配历史记录
  - 重新分配功能

#### 3.3 工作流程自动化

- **自动审核规则**
  - 基于历史数据的信任度评分
  - 新房东 vs 老房东差异化审核
  - 房源质量预评估
  - 自动通过/拒绝规则

## 🔧 技术实现要点

### 前端技术栈

- **Vue 3 + TypeScript** - 类型安全的组件开发
- **Element Plus** - 一致的 UI 组件体验
- **Pinia** - 状态管理优化
- **Vue Router** - 路由权限控制
- **Axios** - HTTP 请求拦截和错误处理

### 后端技术栈

- **Spring Boot** - 微服务架构
- **Spring Security** - 权限控制和认证
- **Spring Data JPA** - 数据访问优化
- **Spring Mail** - 邮件发送服务
- **Redis** - 消息队列和缓存
- **WebSocket** - 实时通知推送

### 数据库设计

- **消息表(messages)** - 站内消息存储
- **通知偏好表(notification_preferences)** - 用户通知设置
- **权限表(permissions)** - 细粒度权限定义
- **审核分配表(audit_assignments)** - 审核员分配记录
