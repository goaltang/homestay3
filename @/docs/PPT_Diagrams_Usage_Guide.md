# 毕业设计答辩 PPT 图表使用指南

## 概述

本指南详细说明了为毕业设计答辩 PPT 创建的 6 个核心技术图表，每个图表都对应 PPT 中的特定页面，用于展示系统的技术亮点和创新点。

---

## 图表文件列表

### 1. **SystemArchitecture.drawio** - 系统架构图

**适用 PPT 页面：** 第 3 页 - 系统架构设计  
**展示目标：** 分层解耦的现代化系统架构

**核心要点：**

- **设计哲学：** Intelligence（智能）+ Security（安全）+ Efficiency（高效）
- **三层架构：**
  - 表示层：Vue 3 技术栈，极致用户体验
  - 服务层：Spring Boot 3 技术栈，核心业务逻辑
  - 数据层：MySQL + Redis，可靠存储与高效访问
- **职责清晰：** 高内聚低耦合，易于扩展维护

**演讲要点：**
"为实现这一目标，我设计了这套前后端分离、分层解耦的系统架构。这种设计保证了各层职责单一、高内聚低耦合，不仅提升了开发效率，更为系统未来的功能扩展与维护打下了坚实的基础。"

### 2. **IntelligentRecommendationMatrix.drawio** - 智能推荐矩阵

**适用 PPT 页面：** 第 4 页 - 核心亮点一：智能驱动  
**展示目标：** 从全局推荐到微观洞察的多层次推荐体系

**核心模块：**

1. **编辑精选 (getRecommendedHomestays)**

   - 动态加权评分模型，融合多维度指标
   - 时间衰减因子保证时效性

2. **热门房源 (getPopularHomestays + diversifyResults)**

   - 热度计算 + 多样性保障算法
   - 打破信息茧房，提升长尾房源曝光

3. **猜你喜欢 (getPersonalizedRecommendations)**

   - 多维度用户画像构建
   - 千人千面的个性化推荐

4. **房源亮点智能提取 (数据驱动 UI)**
   - 前后端协同智能处理：后端数据结构化 + 前端算法计算
   - 自动筛选核心卖点，提升决策效率和转化率

**演讲要点：**
"本项目的第一个核心亮点，是贯穿系统全程的多层次智能推荐矩阵。在宏观层面，我们通过三种不同策略的推荐算法，分别解决了'如何发现好房源'、'如何发现热门房源'和'如何发现我喜欢的房源'这三个问题。而在微观的房源详情页，我们同样采用前后端协同的方式用算法来驱动 UI：后端提供结构化数据，前端进行智能计算，这不仅是简单的功能，更是一种全栈数据驱动的设计思想。"

### 3. **FinancialGradeSecuritySystem.drawio** - 金融级交易安全体系

**适用 PPT 页面：** 第 5 页 - 核心亮点二：坚如磐石的交易安全  
**展示目标：** 三级纵深防御体系解决高并发超卖问题

**防御层次：**

1. **第一级：流量过滤层 (应用层锁)**

   - BookingConflictService + ConcurrentHashMap
   - 内存锁快速拦截 99%并发请求

2. **第二级：数据锁定层 (数据库悲观锁)**

   - @Lock(LockModeType.PESSIMISTIC_WRITE)
   - SELECT ... FOR UPDATE 行级锁

3. **第三级：一致性保障层 (事务隔离)**
   - SERIALIZABLE 最高隔离级别
   - 杜绝所有并发问题

**处理流程：** 1000 个并发请求 → 990 个被应用层过滤 → 10 个进入数据库层 → 1 个成功创建订单

**演讲要点：**
"第二个核心亮点是系统的金融级安全交易机制。为攻克业界常见的'超卖'难题，我设计了这套'应用层过滤、数据库锁定、事务最终保障'的三级纵深防御体系。它不仅解决了问题，更体现了从架构层面系统性应对高并发挑战的设计思路。"

---

### 4. **OrderStateMachine.drawio** - 订单状态机

**适用 PPT 页面：** 第 5 页 - 与交易安全体系配合展示  
**展示目标：** 严谨的订单生命周期管理

**核心机制：**

- 8 个核心状态，12 种转换路径
- isValidStatusTransition 方法严格校验
- 2 个终态保证流程完整性

**状态流转：**
UNPAID → PAID → CONFIRMED → READY_FOR_CHECKIN → CHECKED_IN → COMPLETED

**异常处理：**
支持取消、退款等异常流程，确保业务场景 100%覆盖

**演讲要点：**
"同时，订单的整个生命周期由一个严谨的状态机驱动，确保了业务流程的万无一失。"

---

### 5. **HomestayAuditWorkflow.drawio** - 房源审核工作流

**适用 PPT 页面：** 第 6 页 - 核心亮点三：闭环管理  
**展示目标：** 高效严谨的房源生命周期监控

**关键控制点：**

1. **提交前置校验 (isHomestayReadyForReview)**

   - 房源信息完整性检查
   - 减少 80%无效提交

2. **后台批量处理 (batchReviewHomestays)**

   - 管理员批量审核功能
   - 10 倍效率提升

3. **即时消息通知**
   - 站内信反馈机制
   - 形成管理闭环

**管理闭环：** 提交检查 → 高效审核 → 即时反馈 → 持续优化

**演讲要点：**
"为了从源头保证平台房源的质量，我同样运用状态机模式，设计了这套高效严谨的后台审核工作流。从房东保存草稿，到提交前的前置校验，再到管理员的批量审核和即时反馈，形成了一个完整的房源质量监控闭环。"

## PPT 制作建议

### 时间分配

- **系统架构 (1 分钟)：** 重点强调分层设计思想
- **智能推荐 (1.5 分钟)：** 详细展示四个核心模块
- **交易安全 (1.5 分钟)：** 突出三级防御体系
- **闭环管理 (1 分钟)：** 强调平台治理能力
- **总结展望 (0.5 分钟)：** 技术价值与未来方向

### 演讲技巧

1. **图表配合：** 每个技术点都有对应的可视化图表支撑
2. **层次清晰：** 从宏观架构到微观实现，逻辑递进
3. **突出创新：** 强调技术难点的解决方案和创新思路
4. **数据支撑：** 用具体的性能指标证明技术效果

### 核心亮点总结

- **智能驱动：** 多层次推荐算法 + 数据驱动 UI
- **安全可靠：** 三级纵深防御 + 状态机管控
- **高效管理：** 批量处理 + 闭环反馈

这些图表不仅展示了系统的技术实现，更体现了从问题分析到解决方案的完整技术思路，充分证明了毕业设计的技术深度和创新价值。
