# 房源卡片封面图片优先级修复记录

## 问题描述

用户发现房源卡片显示的是房源图片集中的图片，而不是专门设置的封面图片。房源卡片应该优先显示封面图片（coverImage），而不是图片集（images）中的图片。

## 问题原因分析

在之前的`HomestayCard`组件图片处理逻辑中，优先级设置错误：

**错误的优先级顺序**:

1. 首先检查图片集（images）- ❌ 错误
2. 然后检查封面图片（coverImage）- ❌ 错误

这导致如果房源有图片集，就会显示图片集的所有图片轮播，而不是显示专门设置的封面图片。

## 修复方案

### 修复前的问题代码

```js
const images = computed(() => {
  // 错误：优先使用图片集
  if (
    props.homestay.images &&
    Array.isArray(props.homestay.images) &&
    props.homestay.images.length > 0
  ) {
    return props.homestay.images.map((image: any) => {
      // 返回所有图片，导致轮播显示
    });
  }

  // 其次才使用封面图片
  if (props.homestay.coverImage) {
    return [processImageUrl(props.homestay.coverImage)];
  }

  return [defaultImage];
});
```

### 修复后的正确逻辑

```js
const images = computed(() => {
  const defaultImage = "https://picsum.photos/300/200";

  // 图片URL处理函数
  const processImageUrl = (imageUrl: string): string => {
    // ... URL处理逻辑
  };

  // ✅ 正确：优先使用封面图片（这是房源卡片应该显示的主要图片）
  if (props.homestay.coverImage) {
    return [processImageUrl(props.homestay.coverImage)];
  }

  // ✅ 正确：如果没有封面图片，才使用图片集的第一张图片
  if (
    props.homestay.images &&
    Array.isArray(props.homestay.images) &&
    props.homestay.images.length > 0
  ) {
    const firstImage = props.homestay.images[0];
    if (typeof firstImage === "string") {
      return [processImageUrl(firstImage)];
    }
    return [
      processImageUrl(firstImage?.url || firstImage?.path) || defaultImage,
    ];
  }

  return [defaultImage];
});
```

## 修复效果

### 显示逻辑优化

| 情况                  | 修复前            | 修复后            | 状态 |
| --------------------- | ----------------- | ----------------- | ---- |
| 有封面图片 + 有图片集 | 显示图片集轮播 ❌ | 显示封面图片 ✅   |
| 有封面图片 + 无图片集 | 显示封面图片 ✅   | 显示封面图片 ✅   |
| 无封面图片 + 有图片集 | 显示图片集轮播 ❌ | 显示第一张图片 ✅ |
| 无封面图片 + 无图片集 | 显示默认图片 ✅   | 显示默认图片 ✅   |

### 用户体验改进

- ✅ **一致性**: 所有房源卡片都显示单张封面图片，视觉效果统一
- ✅ **性能**: 减少不必要的图片轮播，提升页面性能
- ✅ **语义化**: 封面图片更能代表房源特色，符合用户期望
- ✅ **加载速度**: 只加载一张图片，减少网络请求

### 技术改进

- ✅ **优先级明确**: 封面图片 > 第一张图片 > 默认图片
- ✅ **逻辑简化**: 单张图片显示，无需复杂的轮播逻辑
- ✅ **兼容性**: 保持对没有封面图片的房源的兼容性

## 相关组件行为

### 模板自动适配

由于现在`images`数组只包含一张图片，模板中的条件判断会自动选择正确的显示方式：

```vue
<template>
  <div class="homestay-image-container">
    <!-- 由于images.length === 1，自动使用单图显示 -->
    <el-carousel
      v-if="images.length > 1"
      height="220px"
      indicator-position="none"
      arrow="hover"
    >
      <!-- 不会执行轮播 -->
    </el-carousel>
    <!-- ✅ 会执行这个分支，显示单张封面图片 -->
    <img
      v-else
      :src="images[0]"
      :alt="homestay.title || homestay.name"
      class="homestay-image"
    />
  </div>
</template>
```
