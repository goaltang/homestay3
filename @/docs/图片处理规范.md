# 房源图片处理统一规范

## 问题背景

在开发过程中发现图片显示不一致的问题，主要原因包括：

1. 前后端图片获取逻辑不统一
2. 图片 URL 格式处理不一致
3. 封面图片 vs 图片集的优先级规则不明确

## 统一规范

### 1. 图片类型定义

- **封面图片 (coverImage)**: 房源的主要展示图片，用于列表页、卡片展示
- **图片集 (images)**: 房源的详细图片集合，用于详情页轮播展示

### 2. 图片获取优先级

**房源主图显示顺序**:

```
封面图片 → 图片集第一张 → 默认占位图
```

### 3. 前端实现规范

#### 3.1 统一工具函数

使用 `@/utils/image.ts` 中的统一工具函数：

```typescript
// 获取房源主图
import { getHomestayMainImage } from "@/utils/image";
const mainImage = getHomestayMainImage(homestay);

// 获取所有图片（去重）
import { getHomestayAllImages } from "@/utils/image";
const allImages = getHomestayAllImages(homestay);
```

#### 3.2 页面使用规范

- **列表页/卡片**: 使用 `getHomestayMainImage()`
- **详情页轮播**: 使用 `getHomestayAllImages()`
- **审核页预览**: 使用 `getHomestayMainImage()`

### 4. 后端实现规范

#### 4.1 统一工具类

使用 `ImageUrlUtil` 工具类处理所有图片 URL：

```java
@Autowired
private ImageUrlUtil imageUrlUtil;

// 处理单个图片URL
String imageUrl = imageUrlUtil.ensureCompleteImageUrl(imagePath);

// 批量处理图片URL
List<String> imageUrls = imageUrlUtil.ensureCompleteImageUrls(imagePaths);

// 获取房源主图
String mainImage = imageUrlUtil.getMainImage(coverImage, images);
```

#### 4.2 DTO 转换规范

在所有 `convertToDTO` 方法中必须使用 `ImageUrlUtil`：

```java
dto.setCoverImage(imageUrlUtil.ensureCompleteImageUrl(homestay.getCoverImage()));
dto.setImages(imageUrlUtil.ensureCompleteImageUrls(homestay.getImages()));
dto.setOwnerAvatar(imageUrlUtil.ensureCompleteImageUrl(homestay.getOwner().getAvatar()));
```

### 5. 配置规范

#### 5.1 服务器 URL 配置

在 `application.yml` 中配置基础 URL：

```yaml
app:
  server:
    url: ${SERVER_URL:http://localhost:8080}
```

#### 5.2 默认图片配置

- 默认占位图路径: `/placeholder-image.jpg`
- 测试图片路径: `/uploads/homestays/fcefb873-2ed7-4284-b1b9-b9145dc2188a.jpg`

### 6. 数据一致性规范

#### 6.1 数据库存储

- 图片路径统一存储为相对路径（以 `/` 开头）
- 完整 URL 由后端动态拼接生成

#### 6.2 API 响应

- 所有图片字段必须返回完整 URL
- 空值处理：`null` 或空字符串统一处理为 `null`

### 7. 测试验证

#### 7.1 前端测试要点

- [ ] 列表页房源卡片显示正确的主图
- [ ] 详情页图片轮播包含所有图片且无重复
- [ ] 封面图片为空时能正确显示图片集第一张
- [ ] 所有图片都为空时显示默认占位图

#### 7.2 后端测试要点

- [ ] 所有 API 返回的图片 URL 都是完整 URL
- [ ] 图片 URL 格式一致（协议+域名+路径）
- [ ] 空值处理正确

### 8. 迁移指南

对于现有代码的迁移：

#### 8.1 前端迁移

1. 导入统一工具函数
2. 替换所有直接访问 `images[0]` 的代码
3. 使用 `getHomestayMainImage()` 替换自定义逻辑

#### 8.2 后端迁移

1. 注入 `ImageUrlUtil` 工具类
2. 替换所有直接拼接 URL 的代码
3. 统一使用工具类方法

- [ ] 是否遵循了图片获取优先级规则
