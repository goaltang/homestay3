# 关键词输入框修复记录

## 问题描述

用户反馈搜索栏关键词输入框存在以下问题：

1. **输入框无法正常输入文字**
2. **输入框外观异常，样式显示不正确**
3. **整体搜索栏布局比例不协调**

## 问题原因分析

### 1. Element Plus 组件冲突

- 使用 `el-input` 组件时，Element Plus 的默认样式与自定义样式产生冲突
- `:border="false"` 属性无法完全移除 Element Plus 的边框和阴影样式
- 复杂的 CSS 覆盖（使用 `!important`）仍然无法完全解决样式冲突

### 2. CSS 样式复杂性

- 需要通过多层嵌套的 CSS 选择器来覆盖 Element Plus 样式
- `.el-input__wrapper` 和 `.el-input__inner` 的样式覆盖不完全

### 3. 布局比例问题

- 关键词输入框空间分配不足（flex: 1.2）
- 与其他搜索项的比例不协调

## 解决方案

### 1. 替换为原生 Input

```vue
<!-- 修改前 -->
<el-input
  ref="keywordInput"
  v-model="searchParams.keyword"
  placeholder="民宿名称、特色..."
  clearable
  :border="false"
  class="keyword-input"
/>

<!-- 修改后 -->
<input
  ref="keywordInput"
  v-model="searchParams.keyword"
  placeholder="民宿名称、特色..."
  class="keyword-input-native"
  @click.stop
/>
```

### 2. 简化 CSS 样式

```css
/* 修改前：复杂的Element Plus样式覆盖 */
.keyword-input {
  border: none !important;
  box-shadow: none !important;
}
.keyword-input .el-input__wrapper {
  background-color: transparent !important;
  border: none !important;
  box-shadow: none !important;
  padding: 0 !important;
}

/* 修改后：简洁的原生样式 */
.keyword-input-native {
  width: 100%;
  border: none;
  outline: none;
  background: transparent;
  color: #717171;
  font-size: 14px;
  padding: 0;
  font-family: inherit;
}
```

### 3. 优化布局比例

- 目的地：`flex: 1.6`（从 1.8 减少）
- 关键词：`flex: 1.4`（从 1.2 增加）
- 日期：`flex: 2.2`（从 2.4 减少）
- 房客：`flex: 0.8`（保持不变）

## 经验总结

### 最佳实践

1. **简单场景优先使用原生元素**：对于基础的文本输入，原生 input 往往比组件库更可控
2. **避免过度依赖 CSS 覆盖**：与其用`!important`强制覆盖，不如选择更合适的基础元素
3. **布局比例要合理测试**：实际内容长度测试比估算更准确

### 技术选择原则

- 功能需求简单时，优先考虑原生元素
- 需要复杂交互时，再选择组件库
- 样式冲突严重时，考虑技术方案替换
