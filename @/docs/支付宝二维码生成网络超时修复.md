# 支付宝二维码生成网络超时问题修复

## 问题描述

在支付页面生成支付宝二维码时出现网络超时错误，导致二维码无法正常生成。

### 错误日志

```
java.net.SocketTimeoutException: Read timed out
```

## 问题原因

1. **支付宝 SDK 默认网络超时设置过短**：默认的连接和读取超时时间不足以应对网络延迟
2. **缺少重试机制**：网络异常时没有自动重试机制
3. **JVM 级别网络配置缺失**：系统级别的网络超时参数未优化

## 修复方案

### 1. 增强 AlipayGateway 网络容错性

**文件**: `homestay-backend/src/main/java/com/homestay3/homestaybackend/service/gateway/AlipayGateway.java`

#### 主要改进：

- **添加重试机制**：支持最多 3 次重试，每次间隔 3 秒
- **智能异常检测**：区分网络超时和其他异常类型
- **配置化重试参数**：重试次数和延迟时间可配置

```java
// 添加重试机制
int maxRetries = paymentConfig.getAlipay().getMaxRetries();
for (int attempt = 1; attempt <= maxRetries; attempt++) {
    try {
        // 支付宝API调用
        AlipayTradePrecreateResponse response = alipayClient.execute(alipayRequest);
        // 处理响应...
    } catch (AlipayApiException e) {
        // 网络超时检测和重试逻辑
        if (attempt < maxRetries && isNetworkTimeoutError(e)) {
            Thread.sleep(paymentConfig.getAlipay().getRetryDelay());
            continue;
        }
        // 返回错误响应
    }
}
```

#### 网络异常检测方法：

```java
private boolean isNetworkTimeoutError(AlipayApiException e) {
    String message = e.getMessage().toLowerCase();
    return message.contains("timeout") ||
           message.contains("connect timed out") ||
           message.contains("read timed out") ||
           message.contains("sockettimeoutexception");
}
```

### 2. 更新 PaymentConfig 配置类

**文件**: `homestay-backend/src/main/java/com/homestay3/homestaybackend/config/PaymentConfig.java`

```java
@Data
public static class AlipayConfig {
    // 原有配置...
    private int connectTimeout = 60000;  // 60秒连接超时
    private int readTimeout = 120000;    // 120秒读取超时
    private int maxRetries = 3;          // 最大重试次数
    private int retryDelay = 3000;       // 重试延迟(毫秒)
}
```

### 3. 添加应用配置参数

**文件**: `homestay-backend/src/main/resources/application.properties`

```properties
# 网络连接配置
payment.alipay.connect-timeout=60000
payment.alipay.read-timeout=120000
payment.alipay.max-retries=3
payment.alipay.retry-delay=3000

# HTTP客户端配置 - 提高网络容错性
spring.http.client.connection-timeout=60000
spring.http.client.read-timeout=120000
spring.http.client.write-timeout=60000
```

### 4. 创建 JVM 级别网络配置

**文件**: `homestay-backend/src/main/java/com/homestay3/homestaybackend/config/NetworkConfig.java`

```java
@Configuration
@Slf4j
public class NetworkConfig implements InitializingBean {

    @Override
    public void afterPropertiesSet() {
        // 设置JVM级别的网络超时参数
        System.setProperty("sun.net.client.defaultConnectTimeout", "60000");
        System.setProperty("sun.net.client.defaultReadTimeout", "120000");

        // 设置HTTP/HTTPS连接超时
        System.setProperty("http.connection.timeout", "60000");
        System.setProperty("http.socket.timeout", "120000");
        System.setProperty("https.connection.timeout", "60000");
        System.setProperty("https.socket.timeout", "120000");

        log.info("网络超时配置完成: 连接超时=60s, 读取超时=120s");
    }
}
```

### 5. 创建 HTTP 客户端配置

**文件**: `homestay-backend/src/main/java/com/homestay3/homestaybackend/config/HttpClientConfig.java`

```java
@Configuration
@Slf4j
public class HttpClientConfig {

    @Bean
    public RestTemplate restTemplate(RestTemplateBuilder builder) {
        RestTemplate restTemplate = builder
            .setConnectTimeout(Duration.ofSeconds(60))
            .setReadTimeout(Duration.ofSeconds(120))
            .build();

        log.info("RestTemplate配置完成，连接超时: 60s, 读取超时: 120s");
        return restTemplate;
    }
}
```

## 修复效果

### 1. 网络容错性提升

- **连接超时**: 从默认值提升到 60 秒
- **读取超时**: 从默认值提升到 120 秒
- **重试机制**: 最多 3 次重试，总计可容忍约 4-5 分钟的网络延迟

### 2. 用户体验改善

- **自动重试**: 网络波动时自动重试，减少用户感知的错误
- **详细日志**: 提供详细的错误信息和重试过程日志
- **友好提示**: 区分网络超时和其他错误，提供针对性的错误提示

### 3. 系统稳定性增强

- **多层次配置**: JVM 级别 + Spring 配置 + 应用配置
- **智能异常处理**: 区分不同类型的异常，采用不同的处理策略
- **配置化管理**: 超时参数和重试策略可通过配置文件调整

## 技术要点

- **重试策略**: 指数退避 vs 固定间隔，当前采用固定间隔 3 秒
- **异常分类**: 区分网络异常、业务异常、系统异常
- **配置优先级**: JVM 系统属性 > Spring 配置 > 应用默认值
- **线程安全**: 重试过程中的线程中断处理
