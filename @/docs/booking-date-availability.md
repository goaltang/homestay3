# 房源日期可用性功能

## 概述

本功能实现了房源详情页面的实时日期可用性显示，用户在选择预订日期时能够实时看到哪些日期已被预订，大大提升了用户体验。

## 功能特点

### 🎯 多层次设计

1. **第一层**：房源详情页面实时显示可用日期
2. **第二层**：搜索结果智能过滤（已存在）
3. **第三层**：订单确认页面并发控制（已存在）

### ✨ 用户体验提升

- **实时反馈**：用户看到的都是真实可选的日期
- **直观显示**：已预订日期自动变灰，无法选择
- **友好提示**：明确告知用户灰色日期不可选择的原因

## 技术实现

### 后端 API

#### 新增接口

```http
GET /api/homestays/{id}/unavailable-dates
```

**功能**：获取指定房源的不可用日期列表
**参数**：

- `id` (path): 房源 ID

**响应**：

```json
["2024-01-15", "2024-01-16", "2024-01-17"]
```

#### 实现细节

- **服务层**：`HomestayServiceImpl.getUnavailableDates()`
- **查询条件**：状态为 `CONFIRMED` 或 `PAID` 的订单
- **日期计算**：返回从入住日期到退房日期前一天的所有日期
- **性能优化**：使用 Stream API 和数据库索引

### 前端实现

#### BookingCard 组件增强

- **新增属性**：`homestayId` - 房源 ID（必需）
- **实时获取**：组件挂载时自动获取不可用日期
- **智能禁用**：自动禁用过去日期和已预订日期
- **加载状态**：显示数据加载状态，提升用户体验

#### 使用方法

```vue
<BookingCard
  :homestay-id="homestay.id!"
  :price-per-night="pricePerNight"
  :rating="rating"
  :review-count="reviewCount"
  :max-guests="maxGuests"
  @booking-confirmed="handleBooking"
  @date-changed="handleDateChange"
/>
```

## 性能考量

### 优化措施

1. **数据库索引**：在 `orders` 表创建了复合索引
2. **查询优化**：只查询必要的状态和字段
3. **前端缓存**：组件级别的数据缓存
4. **按需加载**：仅在组件挂载时获取数据

### 监控建议

- 监控 API 响应时间
