# 订单创建问题调试报告

## 🔍 问题现象

用户在订单确认页面点击"确认预订"时，出现 400 错误：

```
"所选日期已被预订，请选择其他日期"
```

## 🕵️ 问题分析

### 可能原因

1. **数据库中存在冲突订单**：该房源在用户选择的日期已有其他预订
2. **并发冲突检测过于严格**：BookingConflictService 的检查逻辑可能有问题
3. **订单状态问题**：已取消的订单仍被认为是有效预订
4. **时区/日期格式问题**：前后端日期处理不一致

### 检查步骤

#### 1. 检查数据库中的订单数据

```sql
-- 查看特定房源的所有订单
SELECT id, order_number, homestay_id, check_in_date, check_out_date, status, payment_status, created_at
FROM orders
WHERE homestay_id = ?
ORDER BY check_in_date;

-- 查看冲突检测使用的状态
SELECT * FROM orders
WHERE homestay_id = ?
AND status IN ('PENDING', 'CONFIRMED', 'PAID')
AND check_in_date < '用户退房日期'
AND check_out_date > '用户入住日期';
```

#### 2. 检查冲突检测逻辑

当前使用的状态列表：`'PENDING', 'CONFIRMED', 'PAID'`

**问题可能**：

- 已取消的订单(`CANCELLED`)应该不算冲突
- `EXPIRED`过期订单应该不算冲突
- `REJECTED`被拒绝订单应该不算冲突

#### 3. 检查前端发送的数据格式

```javascript
// 确认前端发送的日期格式
{
  "homestayId": 123,
  "checkInDate": "2025-01-18",  // 格式是否正确？
  "checkOutDate": "2025-01-20", // 格式是否正确？
  "guestCount": 2
}
```

## 🛠️ 修复方案

### 方案 1：优化冲突检测逻辑

修改`existsOverlappingBookingWithLock`查询，排除已取消/过期的订单：

```java
@Query("SELECT COUNT(o) > 0 FROM Order o WHERE o.homestay.id = :homestayId " +
       "AND o.status IN ('PENDING', 'CONFIRMED', 'PAID', 'CHECKED_IN') " +
       "AND ((o.checkInDate < :endDate AND o.checkOutDate > :startDate))")
```

### 方案 2：添加更详细的错误信息

在 OrderServiceImpl 中返回更具体的错误信息：

```java
if (hasConflict) {
    // 查询具体的冲突订单
    List<Order> conflictOrders = orderRepository.findConflictingOrders(
        homestayId, checkInDate, checkOutDate);

    String conflictInfo = conflictOrders.stream()
        .map(o -> String.format("订单%s (%s至%s)",
            o.getOrderNumber(), o.getCheckInDate(), o.getCheckOutDate()))
        .collect(Collectors.joining(", "));

    throw new IllegalArgumentException(
        String.format("所选日期与现有订单冲突: %s", conflictInfo));
}
```

### 方案 3：前端预检查

在订单确认页面添加日期可用性检查：

```javascript
// 在用户选择日期后立即检查
async checkDateAvailability(homestayId, checkIn, checkOut) {
  const response = await api.post('/api/orders/check-availability', {
    homestayId, checkInDate: checkIn, checkOutDate: checkOut
  });
  return response.data.available;
}
```

## 🚨 紧急修复

立即可用的快速修复方案：

1. **检查数据库**：手动查看是否有垃圾数据
2. **重启服务**：清理可能的缓存问题
3. **添加日志**：增强错误信息的详细程度

## 📝 测试建议

1. 选择明确没有冲突的日期进行测试
2. 检查具体是哪个房源 ID 出现问题
3. 验证用户权限和登录状态
