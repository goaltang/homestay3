# 支付页面跳转问题修复总结

## 🎯 问题描述

用户反馈在订单确认页面点击"确认预订"后，支付页面无法正常跳转，并且 URL 参数过长导致用户体验不佳。

## 🔍 问题分析

### 1. 主要问题

- **OrderPay.vue 模块加载失败**：`import.meta.env.DEV`在模板中使用导致编译错误
- **订单创建 API 调用失败**：字段名不匹配导致 400 错误
- **URL 参数过长**：通过 GET 参数传递大量数据，URL 变得冗长难看

### 2. 具体错误

```
SyntaxError: "undefined" is not valid JSON
TypeError: Failed to fetch dynamically imported module: OrderPay.vue
[vite] Internal Server Error: import.meta may appear only with 'sourceType: "module"'
```

## 🔧 修复方案

### 1. 修复 OrderPay.vue 编译错误

**问题**：模板中直接使用`import.meta.env.DEV`导致编译错误

**解决**：

```vue
<!-- 修复前 -->
<el-button v-if="import.meta.env.DEV && orderData.status !== 'PAID'">

<!-- 修复后 -->
<el-button v-if="isDev && orderData.status !== 'PAID'">
```

```typescript
// 在script中添加
const isDev = ref(import.meta.env.DEV); // 开发环境标识
```

### 2. 修复订单创建 API 字段名不匹配

**问题**：前端发送的字段名与后端期望的不一致

**解决**：

```typescript
// 修复前
const orderRequestData = {
  totalPrice: orderData.value.totalAmount, // ❌ 后端期望totalAmount
  message: guestInfo.message, // ❌ 后端期望remark
};

// 修复后
const orderRequestData = {
  totalAmount: orderData.value.totalAmount, // ✅
  remark: guestInfo.message, // ✅
};
```

### 3. 优化 URL 参数传递方式

**问题**：URL 过长，包含大量参数影响用户体验

**解决**：使用 Session Storage 替代 URL 参数

#### 修改 useBooking.ts

```typescript
// 修复前：通过URL传递所有参数
router.push({
  path: "/order/confirm",
  query: {
    homestayId: "...",
    checkIn: "...",
    checkOut: "...",
    guests: "...",
    price: "...",
    nights: "...",
    cleaningFee: "...",
    serviceFee: "...",
    title: "...",
    image: "...",
    address: "...",
    hostId: "...",
    hostName: "...",
    autoConfirm: "...",
  },
});

// 修复后：Session Storage + 简化URL
sessionStorage.setItem("booking-details", JSON.stringify(bookingDetails));
router.push({
  path: "/order/confirm",
  query: {
    homestayId: bookingDetails.homestayId.toString(),
    checkIn: bookingDetails.checkInDate,
    checkOut: bookingDetails.checkOutDate,
    guests: bookingDetails.guestCount.toString(),
  },
});
```

#### 修改 OrderConfirm.vue

```typescript
const initOrderData = () => {
  // 优先从session storage获取完整数据
  const storedData = sessionStorage.getItem("booking-details");
  if (storedData) {
    const bookingDetails = JSON.parse(storedData);
    // 构建完整订单数据
    return {
      /* 从session storage构建数据 */
    };
  }

  // 回退到URL参数（兼容旧链接）
  return {
    /* 从URL参数构建数据 */
  };
};

// 页面卸载时清理
onUnmounted(() => {
  sessionStorage.removeItem("booking-details");
});
```

## 💡 最佳实践总结

1. **避免在 Vue 模板中直接使用 import.meta**
2. **前后端 API 字段名保持严格一致**
3. **大量数据传递优先使用本地存储而非 URL 参数**
4. **保持向下兼容性，支持多种数据来源**
5. **及时清理临时存储数据，避免内存泄漏**
