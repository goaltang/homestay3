# 首页设计重构实施记录

## 🎯 重构目标

基于用户反馈"设计有些混乱"，采用**精简整合型方案**重构首页，解决信息层次混乱、功能职责不清、用户认知负担过重的问题。

## 📊 重构前问题分析

### 原始设计的混乱点

1. **功能分散** - 搜索功能分布在 3 个地方

   ```
   搜索栏: [目的地] [日期] [房客] [搜索]
   更多筛选: [关键词] [重置搜索] [搜索]  ← 重复功能
   筛选栏: [房源类型] [价格] [设施] [重置筛选] [应用]
   ```

2. **按钮冗余** - 3 个重置按钮，2 个搜索按钮
3. **状态不清** - 筛选栏始终显示，无论是否在搜索
4. **认知负担** - 用户不知道用哪个功能

## 🛠️ 重构实施方案

### 方案选择：精简整合型

```
🔍 统一搜索栏
[目的地] [入住-退房] [房客] [关键词] [🔍搜索] [↻重置]

🏷️ 智能筛选（搜索后显示）
[全部] [民宿] [酒店] [度假村] [💰价格与设施] [清除筛选]

📋 内容展示
"搜索结果 125个" | "推荐民宿" + [查看全部]
```

## 📝 具体实施步骤

### 第一步：整合关键词搜索到主搜索栏

**变更内容**：

- 在主搜索栏添加关键词输入框
- 移除"更多筛选"区域的重复功能

```html
<!-- 新增关键词搜索项 -->
<div class="search-item keyword-item">
  <div class="label">关键词</div>
  <div class="content">
    <el-input
      v-model="searchParams.keyword"
      placeholder="民宿名称、特色..."
      clearable
    />
  </div>
</div>
```

### 第二步：移除重复的"更多筛选"区域

**删除内容**：

- 整个 `advanced-search` 区域
- 重复的关键词输入框
- 重复的搜索和重置按钮

**结果**：消除了功能重复，简化了界面

### 第三步：优化筛选栏显示逻辑

**变更逻辑**：

```html
<!-- 筛选栏 - 只在搜索后显示 -->
<div v-if="pageState !== 'browse'" class="filter-container"></div>
```

**优化内容**：

- 筛选栏只在搜索状态下显示
- 添加筛选条件计数徽章
- 优化按钮文案："价格与设施"

### 第四步：添加筛选状态指示

**新增功能**：

```javascript
// 活跃筛选条件计数
const activeFilterCount = computed(() => {
  let count = 0;
  if (searchParams.minPrice !== null || searchParams.maxPrice !== null) count++;
  if (searchParams.amenities.length > 0) count++;
  return count;
});
```

**视觉效果**：

- 筛选按钮显示活跃条件数量
- 用户清楚知道当前筛选状态

### 第五步：优化搜索栏布局

**宽度分配优化**：

```css
.search-item.destination-item {
  flex: 1.6;
} /* 目的地 */
.search-item.keyword-item {
  flex: 1.4;
} /* 关键词 */
.search-item.dates {
  flex: 2.2;
} /* 日期(需要更多空间) */
.search-item.guests {
  flex: 1;
} /* 房客 */
```

**布局改进**：

- 根据内容重要性分配空间
- 日期选择器获得更多空间
- 整体布局更加协调

## ✅ 重构成果

### 1. 功能职责清晰化

| 功能区域       | 职责         | 使用场景             |
| -------------- | ------------ | -------------------- |
| **统一搜索栏** | 主要搜索条件 | 用户想要查找特定房源 |
| **智能筛选栏** | 结果优化筛选 | 搜索后进一步缩小范围 |
| **内容展示区** | 结果呈现     | 浏览和选择房源       |

### 2. 用户操作路径简化

**重构前**：

```
输入基础条件 → 点击"更多筛选" → 输入关键词 → 搜索 → 调整筛选 → 应用筛选
```

**重构后**：

```
输入所有条件(包括关键词) → 搜索 → (可选)调整筛选
```

### 3. 界面元素减少

- **搜索按钮**：2 个 → 1 个
- **重置按钮**：3 个 → 2 个（搜索重置 + 筛选清除）
- **功能区域**：3 个 → 2 个

### 4. 状态管理优化

**搜索状态显示**：

- 浏览模式：显示推荐内容，隐藏筛选栏
- 搜索模式：显示结果，展示筛选栏
- 筛选状态：徽章显示活跃条件数量
