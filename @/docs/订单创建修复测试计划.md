# 订单创建修复测试计划

## 问题描述

支付页面无法跳转，根本原因是订单创建失败，返回 400 错误。

## 修复内容

### 1. 字段名不匹配修复

- 前端发送 `totalPrice` → 修改为 `totalAmount`
- 前端发送 `message` → 修改为 `remark`
- 更新 API 类型定义确保一致性

### 2. 修复文件

- `homestay-front/src/views/order/OrderConfirm.vue`
- `homestay-front/src/api/order.ts`

## 测试步骤

### 1. 前端测试

1. 启动前端服务：`npm run dev`
2. 登录系统
3. 选择一个房源进行预订
4. 填写订单确认信息
5. 点击"确认预订"
6. 观察是否成功跳转到支付页面

### 2. 后端测试

1. 检查后端日志中是否有订单创建成功的信息
2. 验证数据库中是否正确创建了订单记录
3. 确认订单状态设置正确

### 3. 网络请求测试

1. 打开浏览器开发者工具
2. 监控 Network 标签页
3. 提交订单时观察 API 请求：
   - POST `/api/orders` 应该返回 201 状态码
   - 响应应该包含订单 ID
   - 无 400 错误

## 预期结果

- 订单创建成功
- 自动确认房源：直接跳转到支付页面
- 房东确认房源：跳转到提交成功页面
- 无控制台错误

## 验证要点

1. ✅ 字段名匹配：totalAmount, remark
2. ✅ API 类型定义一致
3. ⏳ 订单创建成功
4. ⏳ 页面跳转正常
5. ⏳ 支付流程可继续
