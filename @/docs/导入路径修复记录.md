# 导入路径修复记录

## 问题描述

用户报告在开发服务器启动时遇到 Vite 内部服务器错误：

```
Failed to resolve import "@/components/HomestayCard.vue" from "src/views/Home.vue". Does the file exist?
```

## 错误原因分析

在组件抽离重构过程中，我们在`Home.vue`中使用了错误的导入路径：

- **错误路径**: `@/components/HomestayCard.vue`
- **正确路径**: `@/components/homestay/HomestayCard.vue`

HomestayCard 组件实际位于`homestay-front/src/components/homestay/`目录下，而不是直接在`components`根目录。

## 修复过程

### 1. 查找组件真实位置

通过文件搜索工具确认 HomestayCard 组件的实际路径：

```
homestay-front/src/components/homestay/HomestayCard.vue
```

### 2. 修正导入路径

将 Home.vue 中的导入语句从：

```js
import HomestayCard from "@/components/HomestayCard.vue";
```

修正为：

```js
import HomestayCard from "@/components/homestay/HomestayCard.vue";
```

### 3. 组件使用优化

同时将最新上架区域的原始民宿卡片代码也更新为使用 HomestayCard 组件：

```vue
<!-- 修改前：大量重复代码 -->
<div
  v-for="homestay in activeHomestays"
  :key="homestay.id"
  class="homestay-card"
>
  <!-- 复杂的民宿卡片HTML结构 -->
</div>

<!-- 修改后：简洁的组件使用 -->
<HomestayCard
  v-for="homestay in activeHomestays"
  :key="homestay.id"
  :homestay="homestay"
  @click="viewHomestayDetails(homestay.id)"
/>
```

### 4. 路由相关错误修复

修复过程中发现另一个错误：

```
TypeError: Cannot read properties of undefined (reading 'query')
```

**原因**: `handleRouteMessages`方法中重复声明了`useRoute()`
**解决**:

- 在组件顶部添加：`const route = useRoute()`
- 移除`handleRouteMessages`方法内部的重复声明

## 修复结果

✅ Vite 导入错误已解决
✅ 控制台无错误信息
✅ 页面正常渲染
✅ 代码结构更加简洁统一

## 技术细节

### 文件路径结构

```
homestay-front/src/
├── components/
│   ├── homestay/
│   │   └── HomestayCard.vue  ← 正确位置
│   ├── SearchBar.vue
│   └── FilterBar.vue
└── views/
    └── Home.vue
```

### 代码简化效果

- **最新上架区域**: 从 30+行 HTML 代码简化为 1 行组件使用
- **推荐民宿区域**: 已经在之前重构中使用 HomestayCard 组件
- **代码复用**: 所有民宿卡片现在都使用统一的 HomestayCard 组件

## 经验总结

1. **组件导入路径**: 必须与实际文件目录结构保持一致
2. **文件组织**: 将相关组件放在子目录中有助于项目结构清晰
3. **一致性**: 确保所有相同功能的 UI 元素都使用相同的组件
4. **Vue Composition API**: 避免重复声明响应式变量和 composables

## 未来优化建议

- 考虑使用绝对路径别名配置来简化深层目录的导入
- 可以创建组件索引文件来统一管理组件导出
- 建议在大型项目中使用 TypeScript 严格类型检查来提前发现此类错误
