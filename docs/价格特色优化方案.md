# 房源智能特色中价格竞争力显示优化方案

## 问题描述

用户反馈房源详情页的智能特色区域中，预订数据价格竞争力类的特色现在基本上看不到了，基本上都是设施特色。

## 原因分析

### 1. 可比房源数量限制过严

- **原设置**: 最少需要 3 个同类型、同容纳人数（±1 人）的 ACTIVE 状态房源
- **问题**: 测试环境中活跃房源较少，很难达到分析阈值

### 2. 价格竞争力标准过高

- **原标准**: 需要低于平均价格 5%以上才显示"价格优势"
- **问题**: 大多数房源价格接近市场水平，很难触发价格特色

### 3. 市场价格房源无特色显示

- **原逻辑**: MARKET_RATE（±5%范围）和 ABOVE_MARKET 不显示任何价格特色
- **问题**: 占比最大的市场价格房源没有价格相关展示

### 4. 设施特色更容易触发

- **现状**: 房源类型（优先级 11）和设施特色更容易满足条件
- **结果**: 6 个特色位置被设施特色占据，价格特色无法展示

### 5. 数据类型不匹配错误

- **问题**: Repository 查询方法参数类型不一致，部分使用 String status，部分使用 HomestayStatus 枚举
- **错误**: 导致运行时类型转换异常：`Argument [ACTIVE] of type [java.lang.String] did not match parameter type [com.homestay3.homestaybackend.model.HomestayStatus]`

## 优化方案

### 1. 降低可比房源数量要求

```java
// 修改前
private static final int MIN_COMPARABLE_HOMESTAYS = 3;

// 修改后
private static final int MIN_COMPARABLE_HOMESTAYS = 2; // 降低门槛，增加触发概率
```

### 2. 放宽价格竞争力标准

```java
// 修改前的价格等级划分
if (avgDifference <= -5) return COMPETITIVE;     // 需要低于5%
else if (avgDifference <= 5) return MARKET_RATE; // ±5%范围

// 修改后的价格等级划分
if (avgDifference <= -3) return COMPETITIVE;     // 降低到3%，更容易触发
else if (avgDifference <= 8) return MARKET_RATE; // 扩大到±8%范围
```

### 3. 新增市场价格特色显示

```java
case MARKET_RATE:
    return SuggestedFeatureDTO.builder()
            .featureId("PRICE_MARKET_RATE")
            .iconName("TrendCharts")
            .title("市场价格")
            .description("价格符合市场水平，性价比适中")
            .priority(7)
            .build();
```

### 4. 扩大房源匹配范围

```java
// 容纳人数范围：从±1人扩大到±2人
int guestRange = 2;
int minGuests = Math.max(1, homestay.getMaxGuests() - guestRange);
int maxGuests = homestay.getMaxGuests() + guestRange;

// 必要时放松房源类型限制，增加可比房源数量
```

### 5. 修复数据类型不匹配问题

- **修改 PriceAnalysisServiceImpl**: 将 `COMPARABLE_HOMESTAY_STATUS` 从 `String` 改为 `HomestayStatus.ACTIVE`
- **修改 Repository 查询方法**: 统一所有 price 相关查询方法的 status 参数类型为 `HomestayStatus`
- **添加正确的 import**: 确保 `import com.homestay3.homestaybackend.model.HomestayStatus;`

```java
// 修复前
private static final String COMPARABLE_HOMESTAY_STATUS = "ACTIVE";

// 修复后
private static final HomestayStatus COMPARABLE_HOMESTAY_STATUS = HomestayStatus.ACTIVE;
```

### 6. 提高价格特色优先级

```java
// COMPETITIVE 价格特色优先级从8提升到9，超越部分设施特色
.priority(9)
```

## 优化效果预期

### 1. 增加价格特色触发概率

- 可比房源要求从 3 个降到 2 个
