# 订单页面状态分类优化方案

## 问题分析

### 原有设计问题

1. **状态混淆**：将技术层面的订单状态直接暴露给用户
2. **用户视角不清晰**：技术术语如"CONFIRMED"、"PAID"等不符合用户理解
3. **状态重叠**：同一业务状态可能对应多个技术状态
4. **缺少重要分类**：退款相关订单没有单独分类

### 原有状态分类

- 全部订单
- 待确认 (PENDING)
- 已确认 (CONFIRMED)
- 已支付 (PAID)
- 已完成 (COMPLETED)
- 已取消 (CANCELLED)

## 优化方案

### 新的状态分类设计

#### 1. 全部订单 (all)

显示所有订单，不做筛选

#### 2. 待确认 (PENDING)

- **业务含义**：等待房东确认的订单
- **技术映射**：OrderStatus.PENDING
- **用户操作**：取消订单、查看详情
- **时间限制**：24 小时自动取消

#### 3. 待支付 (NEED_PAYMENT)

- **业务含义**：已确认但尚未支付的订单
- **技术映射**：OrderStatus.CONFIRMED + PaymentStatus.UNPAID
- **用户操作**：立即支付、取消订单、查看详情
- **时间限制**：2 小时自动取消

#### 4. 进行中 (IN_PROGRESS)

- **业务含义**：已支付且尚未完成的订单
- **技术映射**：
  - PaymentStatus.PAID
  - OrderStatus.CHECKED_IN
  - OrderStatus.READY_FOR_CHECKIN
- **用户操作**：联系房东、申请退款、查看详情
- **说明**：包含已支付、准备入住、已入住等中间状态

#### 5. 已完成 (COMPLETED)

- **业务含义**：已完成的订单
- **技术映射**：OrderStatus.COMPLETED
- **用户操作**：评价订单、查看详情
- **说明**：可以进行评价的最终状态

#### 6. 已取消 (CANCELLED)

- **业务含义**：各种原因取消的订单
- **技术映射**：
  - OrderStatus.CANCELLED
  - OrderStatus.CANCELLED_BY_USER
  - OrderStatus.CANCELLED_BY_HOST
  - OrderStatus.CANCELLED_SYSTEM
  - OrderStatus.REJECTED
- **用户操作**：查看详情
- **说明**：包含所有取消和拒绝的情况

#### 7. 退款相关 (REFUND_RELATED)

- **业务含义**：涉及退款的订单
- **技术映射**：
  - PaymentStatus.REFUND_PENDING
  - PaymentStatus.REFUNDED
  - PaymentStatus.REFUND_FAILED
- **用户操作**：查看详情、联系客服
- **说明**：独立分类便于用户跟踪退款状态

## 设计特点

### 1. 用户导向

- 按用户实际关心的业务流程分类
- 使用用户易懂的词汇
- 突出用户需要采取的行动

### 2. 状态徽章

- 每个标签页显示对应订单数量
- 便于用户快速了解各状态订单情况
- 非零数量才显示徽章，保持界面整洁

### 3. 智能筛选

- 基于订单状态和支付状态的组合逻辑
- 优先处理退款等特殊状态
- 避免状态重叠和遗漏

### 4. 时间提醒

- 显示待确认和待支付的倒计时
- 帮助用户及时处理重要订单
- 避免订单自动取消

## 用户体验提升

1. **更直观的分类**：用户无需理解复杂的技术状态
2. **清晰的操作指引**：每个状态下明确显示可执行的操作
3. **及时的提醒信息**：倒计时和状态说明帮助用户做决策
4. **完整的状态覆盖**：不遗漏任何订单状态
5. **便捷的数量显示**：徽章数字快速了解各类订单情况

## 对比优势

| 原设计     | 新设计   | 改进效果                 |
| ---------- | -------- | ------------------------ |
| 已确认     | 待支付   | 更明确用户需要的行动     |
| 已支付     | 进行中   | 涵盖支付后的各种中间状态 |
| 无退款分类 | 退款相关 | 独立管理退款流程         |
| 无数量显示 | 徽章显示 | 快速了解各类订单数量     |
| 技术术语   | 业务术语 | 更符合用户理解习惯       |

这个优化方案从用户角度重新设计了订单状态分类，提供了更好的用户体验和更清晰的操作指引。
