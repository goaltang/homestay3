# 后端支付系统

> [!info] 支付模块
> 基于支付宝 API 的支付和退款系统

## 技术架构

| 组件 | 说明 |
|------|------|
| **AlipayGateway** | 支付宝网关封装 |
| **PaymentGateway** | 支付网关接口 |
| **PaymentService** | 支付业务逻辑 |
| **PaymentProcessingService** | 支付处理服务 |
| **AlipayCallbackController** | 支付宝回调处理 |

## 支付流程

```
用户下单 → 创建订单(PENDING)
    ↓
发起支付 → 调用支付宝 API
    ↓
支付宝收银台 → 用户付款
    ↓
支付宝回调 → 更新订单状态(PAID)
    ↓
支付成功 → 发送通知
```

## 退款流程

```
用户申请退款 → 创建退款记录
    ↓
管理员审核 → 调用支付宝退款 API
    ↓
支付宝处理 → 退款回调
    ↓
更新状态 → 订单标记为已退款
```

## 支付配置

```java
// PaymentConfig
appId: String              // 支付宝应用ID
privateKey: String         // 应用私钥
publicKey: String          // 支付宝公钥
gatewayUrl: String         // 网关地址
notifyUrl: String          // 异步回调地址
returnUrl: String          // 同步回调地址
```

## DTO 结构

### PaymentRequest (支付请求)

```java
outTradeNo: String      // 商户订单号
totalAmount: String     // 订单金额
subject: String         // 订单标题
body: String            // 订单描述
```

### PaymentResponse (支付响应)

```java
outTradeNo: String      // 商户订单号
tradeNo: String         // 支付宝交易号
payForm: String         // 支付表单 (网页支付)
qrCode: String          // 二维码 (扫码支付)
```

### RefundRequest (退款请求)

```java
tradeNo: String         // 支付宝交易号
refundAmount: String    // 退款金额
refundReason: String    // 退款原因
```

## 支付状态

```
PENDING (待支付)
    ↓
SUCCESS (支付成功)
    ↓
REFUNDED (已退款)
```

## 核心 Service

| Service | 职责 |
|---------|------|
| `PaymentService` | 支付入口，创建支付请求 |
| `PaymentProcessingService` | 支付处理核心逻辑 |
| `AlipayGateway` | 支付宝 API 调用封装 |
| `OrderLifecycleService` | 订单生命周期管理 |
